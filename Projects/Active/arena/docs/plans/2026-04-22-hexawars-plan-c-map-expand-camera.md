# Plan C: HexaWars Map Expand and Camera

> Explicit-by-contract plan for the builder queue. Runs after Plan A is sealed. No design judgment required at runtime.

## Goal

Grow the playable board from radius 4 (`size = 9`, 61 reachable cells) to radius 7 (`size = 15`, 169 cells), keep `HEX_SIZE = 48` so individual hexes remain readable, and add a viewport camera (pan + zoom) so the spectator can navigate the larger board.

This plan does **not** change the unit roster, abilities, action contract, or item system. Plan A handles unit truth; Plan B adds items.

## Non-Goals

- No engine mechanics changes. No new abilities. No new unit types.
- No new starting positions. Seed positions remain `{q: ±1, ±2, ±3, r: 0}` from the existing unit-spawn rule.
- No fog-of-war reveal radius changes (Plan A already locks scout `vanguard_sight` at radius 2).
- No mobile-specific camera layout. Camera works on desktop and tablet; touch panning is a future plan.

## Open Questions

- None. All decisions in this document are final unless explicitly re-opened by `/qor-plan`.

## Decision Lock-Ins (Builder MUST follow exactly)

- **Board radius** is `7`. Engine `BOARD_SIZE` becomes `15`. The board is the standard hex disc filtered to `max(|q|,|r|,|s|) <= 7`. Cell count is `169`.
- **Hex size** stays `HEX_SIZE = 48` in `src/public/hex-render.js`.
- **Padding** stays `PADDING = 40`.
- **Starting positions** unchanged. The new outer ring (`distance 4..7`) is empty terrain at game start.
- **Demo board literal** is regenerated by a deterministic script (`scripts/generate-demo-board.ts`) using seed `20260422`. The script is committed AND its output is committed. Builder does not hand-write the literal.
- **Zoom range** is `[0.5, 2.0]`. Default zoom on first paint is `1.0`. Wheel zoom step is `0.1`. `+`/`-` keys and on-screen `+`/`-` buttons step by `0.25`.
- **Pan** is mouse-drag (left button) on the SVG, plus arrow keys (`50px` per press) and `WASD` (same). Pan is clamped so the board's bounding box always overlaps at least `200px × 200px` of the viewport — the user cannot pan the board fully off-screen.
- **Reset camera** is a `Home` key, an on-screen `Reset` button, and a double-click on empty SVG background. Reset returns to `zoom = 1.0` and `pan = (0, 0)`.
- **Camera state is per-page-load.** Not persisted across reloads. Not URL-encoded.
- **Touch input** is **not** in scope this plan. Touch handlers are no-ops.

## Phase 2: Demo Board Regeneration Script

### Affected Files

- `scripts/generate-demo-board.ts` — **new**. Deterministic generator for the radius-7 demo board literal.

### Changes

`scripts/generate-demo-board.ts`:

```ts
// Run with: bun run scripts/generate-demo-board.ts > /tmp/demo-board.txt
// The output is a JS literal of the form:
//   board([[q,r,"TERRAIN", control_or_null], ...])
// Cells appear in lexicographic (q, r) order.
//
// Terrain assignment uses a stable seeded PRNG; the seed is hardcoded as 20260422.
// The terrain distribution is exactly:
//   65% PLAINS, 14% FOREST, 9% HILLS, 7% MOUNTAIN, 5% WATER.
// Note: this UI distribution differs from the engine's 70/15/10/5; the demo board
// adds HILLS as a visual variation. The engine never sees this literal.

import { writeFileSync } from "node:fs";

const RADIUS = 7;
const SEED = 20260422;

type Terrain = "PLAINS" | "FOREST" | "HILLS" | "MOUNTAIN" | "WATER";

function makePrng(seed: number): () => number {
  let s = seed >>> 0;
  return () => {
    s ^= s << 13; s ^= s >>> 17; s ^= s << 5;
    return (s >>> 0) / 0xffffffff;
  };
}

function terrainFor(q: number, r: number): Terrain {
  const prng = makePrng(((SEED * 2654435761) ^ (q * 2246822519) ^ (r * 668265263)) >>> 0);
  // discard one to mix
  prng();
  const v = prng();
  if (v < 0.65) return "PLAINS";
  if (v < 0.79) return "FOREST";
  if (v < 0.88) return "HILLS";
  if (v < 0.95) return "MOUNTAIN";
  return "WATER";
}

const cells: string[] = [];
for (let q = -RADIUS; q <= RADIUS; q++) {
  for (let r = -RADIUS; r <= RADIUS; r++) {
    const s = -q - r;
    if (Math.max(Math.abs(q), Math.abs(r), Math.abs(s)) > RADIUS) continue;
    const t = terrainFor(q, r);
    // Initial control: A owns its inner-3 perimeter, B owns the mirror.
    let control: string | null = null;
    if (q <= -3 && Math.abs(r) <= 3) control = "A";
    else if (q >= 3 && Math.abs(r) <= 3) control = "B";
    const ctrl = control === null ? "null" : `"${control}"`;
    cells.push(`[${q},${r},"${t}",${ctrl}]`);
  }
}
process.stdout.write(`/* AUTO-GENERATED */ board([${cells.join(",")}])`);
```

### Affected Files

### Changes

`src/public/camera.js` (new):

```js
const ZOOM_MIN = 0.5;
const ZOOM_MAX = 2.0;
const PAN_MIN_OVERLAP_PX = 200;

let state = { zoom: 1.0, panX: 0, panY: 0 };
const listeners = new Set();

export function getTransform() {
  return `translate(${state.panX}px, ${state.panY}px) scale(${state.zoom})`;
}

export function getState() { return { ...state }; }

export function panBy(dx, dy, viewportRect, contentBox) {
  // Clamp so contentBox never moves so far that less than PAN_MIN_OVERLAP_PX in either axis remains visible.
  // viewportRect: { width, height } of the SVG container.
  // contentBox: { x, y, width, height } of the drawn board AT current zoom.
  // Implementation: compute new pan, then clamp x and y independently.
  state.panX = clampOverlap(state.panX + dx, viewportRect.width, contentBox.width);
  state.panY = clampOverlap(state.panY + dy, viewportRect.height, contentBox.height);
  notify();
}

export function zoomBy(factor, anchor /* { x, y } in viewport coords, optional */) {
  const next = clamp(state.zoom * factor, ZOOM_MIN, ZOOM_MAX);
  if (anchor && next !== state.zoom) {
    // Keep the anchor point stationary in viewport coords across the zoom change.
    const k = next / state.zoom;
    state.panX = anchor.x - k * (anchor.x - state.panX);
    state.panY = anchor.y - k * (anchor.y - state.panY);
  }
  state.zoom = next;
  notify();
}

export function reset() {
  state = { zoom: 1.0, panX: 0, panY: 0 };
  notify();
}

export function subscribe(fn) {
  listeners.add(fn);
  return () => listeners.delete(fn);
}

function notify() { for (const fn of listeners) fn(getState()); }
function clamp(v, lo, hi) { return Math.min(hi, Math.max(lo, v)); }
function clampOverlap(p, viewSize, contentSize) {
  // Largest p (most positive) where the right/bottom edge of content still overlaps by PAN_MIN_OVERLAP_PX with viewport
  const maxP = viewSize - PAN_MIN_OVERLAP_PX;
  // Smallest p (most negative) where the left/top edge of content still overlaps
  const minP = PAN_MIN_OVERLAP_PX - contentSize;
  return clamp(p, minP, maxP);
}
```
