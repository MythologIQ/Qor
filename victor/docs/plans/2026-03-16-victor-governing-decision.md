# Victor Governing Decision

## Status

Accepted

## Decision

Victor is the resident semantic authority for Zo-Qore.

Builder Console is the governed execution and operational mediation subsystem within Zo-Qore, not the canonical semantic authority.

Victor may consolidate, retrieve, reflect on, and propose actions from workspace knowledge, but any governed write or automation that changes tracked project state must remain auditable and pass Builder governance when operating through Builder surfaces.

## Why

The system needs one explicit semantic authority to avoid split-brain behavior.

Victor exists to turn chaotic input into grounded memory, structured recall, and governed next-step proposals. That role requires semantic continuity, provenance-first memory, contradiction preservation, and inspectable retrieval.

Builder Console exists to govern execution, planning state changes, automation history, and promotion surfaces. That role requires bounded writes, policy mediation, and reviewable operational traces.

If Builder quietly owns semantic authority while Victor appears primary, the system will drift into hidden state ownership and unverifiable recall. If Victor owns execution directly without Builder governance, the system will outrun its trust boundary.

## Governing Constraints

- Victor remains observe-first by default.
- Vector similarity may trigger recall, but graph-linked evidence and governance decide what is trusted.
- CAG is short-term memory only and may not become durable memory without consolidation and governance.
- Contradictions are preserved as friction, not flattened for convenience.
- Failure memory is durable negative learning and must influence future recall and write decisions.
- Builder governance is binding for Builder-mediated writes, task status changes, and governed automation.
- Promotion from reflection to supervised automation to higher autonomy tiers requires explicit verification evidence and promotion review.

## Operational Consequences

- New ingest should preserve provenance, epistemic type, and governance state on memory artifacts.
- Retrieval should prefer grounded recall and explicitly downgrade to advisory or blocked when evidence is incomplete or constrained.
- Self-reflection is allowed as a read-oriented activity and should produce inspectable outputs.
- Automation proposals are valid only when their grounded context passes governance and remains linked to durable evidence.

## Current Application

This decision governs:

- Victor memory kernel behavior
- research corpus ingest and consolidation
- heartbeat reflection
- governed automation previews
- future autonomy promotion review
